# ERC20 Token Specification (Kiro Spec)

## ChainSpec Metadata
*   **Contract Name**: `MyGameToken`
*   **Symbol**: `MGT`
*   **Standard**: `ERC20`

## Security Requirements
1.  **Access Control**: Only the `owner` address can mint new tokens.
2.  **Pausable**: The contract owner can pause all transfers in case of emergency.
3.  **Supply Cap**: The total supply can never exceed 1,000,000 tokens.

## Functions

### `mint(address to, uint256 amount)`
*   **Access**: `onlyOwner`
*   **Preconditions**: 
    *   `totalSupply + amount <= 1,000,000` (Cap check)
*   **Postconditions**:
    *   `balanceOf(to)` increases by `amount`
    *   `totalSupply` increases by `amount`

### `burn(uint256 amount)`
*   **Access**: Any user
*   **Preconditions**:
    *   `balanceOf(msg.sender) >= amount`
*   **Postconditions**:
    *   `balanceOf(msg.sender)` decreases by `amount`
    *   `totalSupply` decreases by `amount`
